# Claude in Chrome 向け指示書 C — 今回の修正箇所ピンポイント確認

このセッションで修正した18ファイル（`feat/vue3-migration`ブランチ）+ 1ファイル（別ブランチ
`feat/fix-map-options-icon-alignment`）が、それぞれ期待通りに動作するかを確認する指示書。
ログイン不要。

## 前提

- `feat/vue3-migration` ブランチで `npm run dev` を起動し、`http://localhost:5173/` へアクセス
- 各項目の「Before」は修正前の壊れていた挙動、「期待結果」が確認すべき内容

## 項目一覧

### 1. b-loadingのスピナー表示（10箇所、`:model-value`へ修正）

**Before**: ローディングスピナーが一切表示されなかった（`active` propがBuefy3に存在しないため）

- `/map`: フィルタ操作中またはスタイル未取得の間、地図全体にローディングオーバーレイが出るか
- `/spots/sotawatch`: 初回ロード中に `b-loading` のスピナーが一瞬見えるか（Spot件数が多いと一瞬で消えるため、リロードを2〜3回試す）
- `/spots/rbn`: 同上
- `/activators/<callsign>`（例: `/activators/DL6FBK`）: ページ内の複数箇所（Spots/RBN Spots読み込み中、Activations読み込み中）でスピナーが出るか。初回ロード直後を目視で確認
- Summit詳細ページのMiniMap（例: `/summits/JA/NN-001`）: 地図タイル読み込み中にスピナーが出るか

### 2. NavBarのバーガーメニュー（`v-model`へ修正）

**Before**: モバイル幅でバーガーメニューを開き、ナビリンクをタップしてもメニューが開いたままだった

- ブラウザ幅をモバイル幅（375〜414px程度）にリサイズ
- 右上のハンバーガーアイコンをタップ→メニューが開く
- メニュー内の「Summits」等のリンクをタップ→ページ遷移**かつ**メニューが自動的に閉じることを確認

### 3. モバイルカードリストのページネーション（CardPagination、`v-model`へ修正）

**Before**: モバイル幅のカード一覧でページ送りボタンをクリックしても何も起きなかった

- モバイル幅のまま `/activators/DL6FBK`（19件のスポット履歴あり）を開く
- 画面を下にスクロールし「Recent spots」等のカードリストのページ送り（次へ/数字）をクリック
- カードの内容が実際に切り替わることを確認

### 4. EditAlertのFrequency-Mode(s)欄（`confirm-keys`へ修正）

**Before**: この欄でキーを1つ押すたびにJavaScriptエラー（コンソールに`TypeError`）が発生していた

- `/alerts` → 右上「Add」ボタン → EditAlertモーダルを開く
- 「Frequency-Mode(s)」欄に `7.030-cw` と入力しEnter（またはカンマ）→タグとして確定されるか
- ブラウザのDevToolsコンソールを開き、入力中に赤いエラーが出ないことを確認
- カンマ区切りで複数一度に入力（例: `7.030-cw, 14.250-ssb`）→2つのタグに分割されるか
- スペースキーでもタグが確定するか（`7.030-cw` と入力後、スペースキーを押す）

### 5. SummitPopupの表示幅（`max-width`を追加）

**Before**: 地図でサミットをクリックした際のポップアップが、常にMapLibreのデフォルト240px幅に固定されていた（意図は600px）

- `/map` で任意のサミットマーカーをクリックしてポップアップを開く
- ポップアップの横幅が240pxより明らかに広い（600px相当）ことを目視確認
- 特に写真付きサミット・長いサミット名のものだと違いが分かりやすい

### 6. アイコンの位置ずれ（`feat/fix-map-options-icon-alignment`ブランチ、10セッション持ち越しの既知バグ）

**⚠️ このブランチだけ別**: ホスト側で `git checkout feat/fix-map-options-icon-alignment && npm install && npm run dev` してから確認してください（`feat/vue3-migration`とは別ブランチです）

**Before**: `/map`左上、上から2番目のアイコン（地図オプション）をクリックすると、アイコン自体が
パネルの下端まで（約345px）移動してしまい、パネルを閉じるまでアイコンが見えなくなっていた

- `/map`で左上の地図アイコン（フィルタアイコンの下）をクリック
- パネルが開いた後も、**アイコン自体の位置が変わらない**（パネル上部に留まる）ことを確認
- もう一度クリックしてパネルを閉じ、アイコンの位置が最初と同じであることも確認

### 7. デッドコード削除（実質確認不要、念のため）

- `SwisstopoInfo`/`BasemapAtInfo`: Switzerland/Austriaの座標をブラウザの位置情報で偽装しない限り表示されないポップアップのため、通常は再現できません。確認できなくて問題ありません
- `SummitVideosGroup`の`webp`削除: 動画付きサミットページ（例: YouTube動画があるサミット）で動画のサムネイル表示が変わらないことだけ確認できれば十分です
