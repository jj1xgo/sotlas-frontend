# Claude in Chrome 向け指示書 E — 全体回帰確認（地図ページ）

地図関連（`/map`とその周辺）の回帰確認。今回のセッションでTurnstile devバイパスの移植・
`Map.vue`のb-loading修正・`SummitPopup`のmax-width修正・アイコン位置ずれ修正を行ったため、
地図まわりが全体的に壊れていないかを確認する。ログイン不要。

## 前提・注意事項

1. **2つのブランチにまたがります**:
   - 通常の項目（1〜7）: `feat/vue3-migration` ブランチ
   - アイコン位置ずれ関連（8）: `feat/fix-map-options-icon-alignment` ブランチ（別ブランチ、
     `git checkout feat/fix-map-options-icon-alignment && npm install && npm run dev`）
2. 本番のCloudflare Turnstileは通常表示されます（コンテナ内検証で使った開発用バイパスは
   `VITE_MAPTILER_DEV_KEY`環境変数がある場合のみ有効で、通常のホスト側`npm run dev`では
   無効です）。人間であることの確認が必要な場合があります
3. 想定内のノイズ: `api.maptiler.com`のカスタムスタイル404（標準スタイルへ自動フォールバックする
   ため実害なし）、`en.wikipedia.org`関連のネットワークエラー

## 項目一覧（`feat/vue3-migration`ブランチ）

### 1. 地図の基本表示

- `/map` にアクセス→地図タイルが表示されるか（Turnstile通過後）
- ズーム・パン操作（マウスホイール、ドラッグ）が機能するか
- 右上のズームボタン（+/-）が機能するか

### 2. 左上のコントロール群

- フィルタアイコン（漏斗マーク）クリック→パネル開閉
- 地図オプションアイコンクリック→パネル開閉、**チェックボックス（Regions/Contour
  lines/Hillshading等）をON→ページリロード→設定が保持されているか**
- ダウンロードアイコンクリック→`map.png`という画像ファイルがダウンロードされるか
  （メニューは出ません、クリック即座にダウンロードが始まる仕様です）

### 3. サミットポップアップ

- 地図上のサミットマーカーをクリック→ポップアップが開く
- 写真・標高・ポイント・アクティベーション回数等が表示されるか
- 「Close」「Minimize」ボタンが機能するか
- ポップアップの幅が指示書Cの項目5で確認した600px相当になっているか

### 4. 現在地・その他コントロール

- 右側のGeolocateControlボタン（現在地取得）
- 地図を右クリック→座標情報ポップアップが出るか
- 右側の描画ツール（線を引く、削除する）

### 5. サミットURL直接アクセス

- `/map/summits/JA/NN-001` のようなURLに直接アクセスして、該当サミットにフォーカス・
  ポップアップ表示されるか
- `/map/regions/JA/NN` のようなURLで、該当リージョンにズームされるか

### 6. Summit詳細ページのMiniMap

- `/summits/JA/NN-001` を開き、右側のMiniMap（小地図）が表示されるか
- ズームボタン・拡大ボタンが機能するか

### 7. モバイル幅での地図表示

- ブラウザ幅をモバイル幅（375〜414px）にリサイズし `/map` を開く
- レイアウトが大きく崩れていないか
- 右下の帰属表示（© MapTiler © OpenStreetMap）が正しく表示されるか（コンパクト化はされない
  仕様と確認済みなので、フルテキスト表示のままで正常です）

## 項目（`feat/fix-map-options-icon-alignment`ブランチ）

### 8. アイコン位置ずれ修正の確認

上記1〜7のうち、特に「2. 左上のコントロール群」の地図オプションアイコンについて、
指示書Cの項目6と同じ内容を再確認してください（パネルを開いてもアイコンの位置が変わらないこと）。

## 完了後

NG項目があれば、URL・操作手順・実際に起きたこと（スクリーンショット・コンソールエラー）を
添えて報告してください。全てOKであれば「全項目OK」とだけ報告して構いません。
