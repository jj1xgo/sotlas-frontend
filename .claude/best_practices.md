# Best Practices

> `lessons.md` の 13 項目を蒸留した高レベル原則集。
> 詳細な教訓・事例は `lessons.md` を参照。
> グローバル層（`~/.claude/best_practices.md`）収載済みの原則は再掲せず、
> 本ファイルにはプロジェクト固有の具体化のみを置く。

## 1. 【fork 状態は実データで】

**何をするか**: fork/upstream の同期・マージ・push 状態は、patch-id 比較や handover の記載でなく、実データ（`git diff A B -- <path>` の内容 diff、`git fetch` + 実リモート照合）で確認する。特に rebase 等の履歴書き換えを挟んだブランチは記載が古くなりやすい。

**なぜ有効か**: patch-id 比較の誤判定（1コミット複数目的）と handover の「未push」記載の食い違い（旧系譜がリモートに残存）を、実データ照合でどちらも検知できた。

**よくある失敗パターン**: コミット単位のツール出力や引き継ぎ記載を鵜呑みにして同期状態を断定する。

## 2. 【upstream を先に確認】

**何をするか**: 「これはこの fork だけのローカル事情」と断定する前に、upstream の履歴・PR（`git log --all`、`git branch --contains`）を確認する。ローカルパッチを説明するドキュメント記述は、実装ファイルの中身と定期的に突き合わせる。

**なぜ有効か**: 「ローカルパッチ」と思っていた FontAwesome 対応が実は upstream にマージ済み（PR #43）で、fork 側が同期していなかっただけと判明。ドキュメントと実装の乖離が原因誤伝の元だった。

**よくある失敗パターン**: CLAUDE.md 等の記述だけを根拠に実装の現状を説明する。

## 3. 【動作検証は実経路で】

**何をするか**: 「このライブラリは Vue 3 / Vite で動くか」の検証は Node 単体スクリプト（`require`/`import` が例外を投げないことの確認）で完了とせず、`npm run build`・実ブラウザ（ホスト側 `npm run dev`）まで通す。Node の CJS/ESM 解決は Vite の実バンドル解決経路と異なる。

**なぜ有効か**: Node 単体検証ではライブラリの `install()` 本体に到達しない偽陰性が生じ、白画面（全ページ描画停止）をユーザー報告で初めて検知した。実バンドラ経路まで通す検証を完了条件にすることで、この種の偽陰性を出荷前に捕捉できるようになった。

**よくある失敗パターン**: Node スクリプトが例外を投げなかったことを「実ブラウザでも動く」と読む。

## 4. 【検証は観測可能性込みで】

**何をするか**: 検証項目を計画に挙げる際は、その変化が人間の目視・短時間の観測で判別できるか（観測可能性）を先に見積もり、低視認性なら定量根拠付きで事前に説明する。

**なぜ有効か**: 太陽の見かけの経度移動は約0.25°/分で、「1分ティックでの移動確認」は目視では判別不能なのが仕様。事前説明があれば「動いていない＝バグ」という誤読を防げる。

**よくある失敗パターン**: 実行可能かだけを見て、観測できるかを見ずに検証項目を立てる。

## 5. 【描画の暗黙デフォルトを明示】

**何をするか**: 描画ライブラリの省略時デフォルト（例: MapLibre の `fill` レイヤーは `fill-outline-color` 未指定だと `fill-color` と同色でアウトラインを描く）が絡む見た目のアーティファクトは、明示指定（透明化等）で潰す。半透明ポリゴンの重ね合わせでは特に注意する。

**なぜ有効か**: 縮退辺を持つポリゴンで自動アウトラインが重なり線状アーティファクトになった。明示的な透明化で解消し、以後の重ね描画実装の定石になった。

**よくある失敗パターン**: アーティファクトの原因を自分のジオメトリ生成側だけに探し、レンダラの暗黙仕様を疑わない。

## 6. 【許可リストは実走で確定】

**何をするか**: allowed-domains 等の環境許可リストは、`package.json` の依存を全種類（registry / `github:` / `git+https:` 等）洗い出して設計し、実際にコンテナ内で `npm ci` が通るまで完成としない。

**なぜ有効か**: `github:` 形式の依存が `codeload.github.com` から tarball を取ることを、registry.npmjs.org だけ許可して実走するまで見落としていた。実走確認を完成条件にして検知できた。

**よくある失敗パターン**: 依存の取得経路を registry 一種と仮定して許可リストを机上で確定する。
